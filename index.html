Importantü§î Discussion Questions: Stage 3 - Structuring of Visual Mappings

Question 1: Geometry Choices - Why is a horizontal bar chart better than a scatter plot for this data? - When would you choose a vertical bar chart over horizontal?

Question 2: Aesthetic Mappings - What does the color highlighting accomplish in Approach 4? - How does position (x/y) compare to color for encoding data?

TipSection 3 Answers: Structuring of Visual Mappings

Answer 1 Geometry Choices: - A horizontal bar chart is better than a scatter plot for this data because it is easier to compare the runs per game for each stadium. The stadium names are categorical and often long, so placing them on the y-axis keeps the labels readable. The bar lengths clearly show differences in average runs, making comparisons straightforward, while a scatter plot would be less effective for displaying one numeric variable across categories. - You would choose a vertical bar chart over horizontal when you want to compare the runs per game for each stadium and the stadiums are not too many. A vertical bar chart is better when you have a small number of categories and want to compare their values.

Answer 2 Aesthetic Mappings: - The color highlighting draws attention to specific stadiums or patterns in the data. In this case, it accomplishes that Colorado is the most run-friendly ballpark in 2010. - Position (x/y) is the most accurate way to show numerical differences because people can easily compare how high, low, long, or short something is on a chart. It‚Äôs best for showing quantitative values like average runs per game. Color, on the other hand, is better for showing categories or groups and for drawing attention to certain data points. It helps the viewer quickly see patterns or differences, but it‚Äôs less precise for comparing exact values.

# Stage 4: Formatting for Your Audience
# Create a professional, publication-ready visualization

# Set style for professional appearance
plt.style.use("seaborn-v0_8-whitegrid")

# Create the main visualization
fig, ax = plt.subplots(figsize=(8, 6))

# Create color array for highlighting Colorado
colorado_colors = ["darkorchid" if stadium == "COL" else "lightgrey" 
                   for stadium in avgDF_2010_sorted.home]

# Create horizontal bar chart
bars = ax.barh(avgDF_2010_sorted.home, avgDF_2010_sorted.totalRuns, color=colorado_colors)

# Add title and labels
ax.set_title("Colorado (COL) is the Most Run-Friendly Ballpark in 2010", 
             fontsize=16, fontweight='bold', pad=20)
ax.set_xlabel("Average Runs Per Game", fontsize=12)
ax.set_ylabel("Stadium (Home Team)", fontsize=12)

# Add legend
colorado_bar = plt.Rectangle((0,0),1,1, color="darkorchid", label="Colorado Rockies")
other_bar = plt.Rectangle((0,0),1,1, color="lightgrey", label="Other Stadiums")
ax.legend(handles=[colorado_bar, other_bar], loc='lower right', frameon=True)

# Add annotation for Colorado
colorado_index = avgDF_2010_sorted[avgDF_2010_sorted.home == "COL"].index[0]
colorado_runs = avgDF_2010_sorted[avgDF_2010_sorted.home == "COL"]["totalRuns"].iloc[0]
ax.annotate(f"COL: {colorado_runs:.2f} runs/game", 
            xy=(colorado_runs, colorado_index), 
            xytext=(colorado_runs + 0.5, colorado_index),
            arrowprops=dict(arrowstyle='->', color='darkorchid', lw=2),
            fontsize=10, fontweight='bold', color='darkorchid')

# Set x-axis to start from 0 for better comparison
ax.set_xlim(0, max(avgDF_2010_sorted.totalRuns) * 1.1)

# Add grid for easier reading
ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

# Print summary statistics
print(f"\nSummary Statistics for 2010:")
print(f"Colorado (COL) average runs per game: {colorado_runs:.2f}")
print(f"League average runs per game: {avgDF_2010_sorted.totalRuns.mean():.2f}")
print(f"Colorado is {((colorado_runs / avgDF_2010_sorted.totalRuns.mean()) - 1) * 100:.1f}% above league average")
